# 见微路线图

路线图表达优先级，不是交付日期或永久承诺。第三方平台接口、许可证和维护状态变化时，顺序会调整。

## 近期：让第一次使用更可靠

- 在全新机器上验证一键 Docker 安装。
- 为未配置、授权过期、额度不足和上游异常提供更明确的状态与处理建议。
- 提供脱敏演示数据和产品截图，不要求新用户先配置所有平台。
- 完善版本升级、数据库迁移和回滚说明。
- 增加关键采集器的健康状态、延迟和失败原因统计。

## 下一阶段：让信息更值得读

- 改进低成本模型的结构化输出校验与自动修复。
- 优化中文标题、摘要和推荐理由的一致性。
- 支持用户管理自己的主题和标签体系。
- 为精选、最新和全部信息提供更清楚的排序解释。
- 增加内容处理成本和调用量的可视化预算。

## Provider 扩展

- 保持 X、微信、搜索和 RSS 采集器的可替换接口。
- 增加新 Provider 前先明确授权方式、费用、频率和失败边界。
- 优先支持官方接口和用户自主提供的服务。
- 非官方路径必须可关闭、可退避、可观察，不能伪装成永久稳定能力。

## 暂不承诺

- 绕过平台风控或无限制抓取。
- 为第三方登录、Cookie、会员权益和 API 价格提供稳定性保证。
- 多租户 SaaS、公开注册和商业计费。
- 替用户判断采集行为是否满足其所在地区或组织的法律要求。

欢迎通过功能建议模板说明具体场景。路线图会优先处理能够改善多数自托管用户体验、且不会扩大账号与合规风险的问题。
