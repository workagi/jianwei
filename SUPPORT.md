# 获取帮助

## 适合提交 Issue 的情况

- 在最新 `main` 或最新发布版上可以稳定复现。
- 一键启动、管理后台、采集调度、去重、筛选或展示行为与文档不符。
- 你能够提供脱敏后的日志、版本和复现步骤。
- 你提出的是清晰、可验证的改进建议。

提交前请先运行：

```bash
./start.sh doctor
pnpm audit:open-source
```

## 通常不属于项目缺陷

- X、xAI、微信、搜索服务或模型服务调整了接口、额度、价格或风控。
- SuperGrok 会员权益没有转化为 API 额度。
- WeRSS 或微信扫码状态过期。
- 某个公众号文章被删除、限制访问或没有可提取正文。
- 用户主动配置了过高频率、过大并发或错误的 Base URL。
- 把 WeRSS、TrendRadar、数据库或其他管理服务直接暴露到公网后产生的问题。

这类情况仍可以在 Discussion 或普通 Issue 中交流经验，但维护者无法承诺恢复第三方能力。

## 提交信息

请提供：

- 见微版本或提交 SHA。
- 操作系统、CPU 架构和 Docker 版本。
- 使用 `./start.sh` 还是生产 Compose。
- 哪个信息源、Provider 和监控类型出现问题。
- 脱敏后的错误文本和最小复现步骤。

请删除截图和日志里的 API Key、Cookie、Access Key、二维码、内部域名、真实账号和私人内容。

安全问题请改用 [SECURITY.md](SECURITY.md) 中的私下报告渠道。
